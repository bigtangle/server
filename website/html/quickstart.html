Quick Start for BigTangle Wallet after Download

Payment
You must have tokens to send a payment to other addresses. You can select a token from the list and enter an amount and address to pay. Then, after clicking the button Send, the system will allow you to save the address in the contact list. Next time, you can select the address from this list. If you cannot find the payment in the balance, please check the Blocks window. There may be a delay in the range of seconds for payment confirmation. The column pending will indicate with an asterisk if a payment is still pending.
Tokens
Tokens can be anything from fiat money to stocks of companies. Anyone can issue any number of tokens. As an example, you can create your work days as token for tracking and trading. Click the create button in the token window.
Enter the name of your token: My Work Days
Enter the amount of tokens: 365
Select an address as the issuer and id of this token.
Finalize this.
You may click search to refresh your balance and check the block evaluations in the window Blocks if your token is not shown in your balance.
Markets
Markets can be stock exchanges for stocks, bonds etc. Financial products can easily be traded by creating new token equivalents. The system has an integrated free market for token exchanges. In the last step we created a work day token. Now we want to get system coins in exchange for our work days. Click the order tab in the Markets window. Select your token and give a price for your work days. Then wait for someone to be matched to your sale. For testing purposes, we have an agent running as an automated service: The agent will offer to buy any tokens posted for sale.
Exchange
When an order is matched, it becomes an offer (transaction) that can be signed in the wallet. In the Exchange window you will find the offer and it is already signed by the automated agent. Click the sign button and the deal is finalized such that you will trade the tokens. You can also create a custom transaction offer. To do so, click create in the Exchange window.
Blocks
By default it will display blocks related to you only. If you perform transactions but they do not show up in the main window balance, you will probably find the blocks here with ratings lower than 75 and milestone set to false. These blocks are not confirmed yet due to conflicts, insufficient processing time or orphaning. You can select the blocks and view their details.
Multi Sign
It is possible to set payments to require N of M signatures. For example, two signatures might be required to authorize a transaction. To do so, the first key holder signs by click export, then sends it to the next key holder. The next key holder imports the block file and clicks the send button to perform the payment.
Settings
The BigTangle wallet is a key store. It saves all your public and private keys in your wallet. You can set a password to protect your wallet, but IT IS IMPOSSIBLE TO RECOVER THE KEYS WITHOUT YOUR PASSWORD. Your wallet is per default saved in your home directory with username.wallet as filename, e. g. C:\Users\username\username.wallet. Your contact list and your watched token list files are in the same directory.
Test Clusters
There are two server clusters for the current Testnet operated by us. One of the clusters is in China and can be connected to via cn.server.bigtangle.net, the other is in Europe and is found under de.server.bigtangle.net. The wallet will connect to the server in China if your operations system is Chinese. The clusters are currently connected peer to peer. There will be more clusters in the test network after release of the BigTangle node.
